Integration Overview

From a high-level, there are three components of a LevelUp Point of Sale integration:

1. Authentication

  • Authenticate with the LevelUp API and obtain an OAuth token using the /oauth/access_tokens.json endpoint.
  • Retrieve information about the currently logged in user using the /users/:id.json endpoint.
  • Save the access_token, merchant ID and location ID to use with all transactions.
  • Discard the merchant login credentials.

2. Orders & Refunds

  • Scan a consumer’s QR code from the screen of a consumer’s smartphone running the LevelUp app. Send the QR code and order details to LevelUp using the /merchants/:id/orders.json endpoint.
  • Allow for refunds of the payment through the /orders/:order_id/refund.json endpoint.

3. Analytics

  • Apply the payment to the check and the POS accounting ledger.
  • Close out the check

Schema

All communication takes place over HTTPS, and the server responds with JSON messages described in more detail below. The API is currently at version 13, and all endpoints should be prefixed with https://api.thelevelup.com/v13/.

For POST requests, parameters not included in the URL should be encoded as JSON with a Content-Type of application/x-www-form-urlencoded.

Client Errors

The API responds to errors with an appropriate HTTP status code and (optional) error message.

{"error": "Sorry, you can not leave any fields blank"}

The login /oauth/access_tokens endpoint will return OAuth 2.0 error values in the error attribute; human-readable errors there are returned in the error_description attribute instead.

{“error_description”: “Bad email or password”, “error”: “invalid_grant”}
